Design Science Research (DSR) is a research methodology that revolves around the solution of practical problems through the creation of purposeful artefacts. The final goal is to generalize the solution to these practical and situated problems in order to be able to solve similar ones. Thus, an initial exhaustive analysis of the problem becomes crutial. That is why DSR projects tend to start with the Problem Analysis, in which problems and their surrounding contexts are analyzed.

But, first of all, what is meant by a problem?
A problem is defined as the difference between the current state and a desired state.
As said previously, the Problem Analysis is the first phase of a DSR project. The success of the project relies largely on a deep understanding of the problem to be addressed. This phase can be seen as a process in which a vague statement of a problem is transformed into a matured analysis, in which both the problem and its context are described.

The problem analysis comprises three different activities: define precisely, position and justify and find root causes. Next subsections explain each of them in detail.
A Problem Analysis starts by a precise definition of the problem. A precise definition helps to limit the scope of the project, hence increasing the chances of coming up with a right solution. This definition must also be as clear as possible, as it has to be understood by different people. Different stakeholder types can be engaged in the formulation of the problem in order to integrate their points of view and knowledge into a more complete definition.
The problem is defined in the highlighted branch of the mind map.
Problems do not exist in an isolated way but in a context, and it has to be well understood in order to address the problem effectively. Therefore, the practice in which it appears needs to be described by analizing the stakeholders, activities, environment, purpose and tooling. Additionaly, the significance of the problem within the practice has to be justified in order for people to agree that it worths solving.
The highlighted branch of the mind map is the place for positioning and justifying the problem.
When anaysing a problem, not only is it necessary to understand the context in which it takes place, but also it is important to find which are the causes that are beneath it. In this sense, the Root Cause Analysis is a technique for uncovering those underlying causes also known as root cuases. These causes are the ones to be tackled if we want to effectively address the problem.
There are different approaches for representing causal models. We resort to an adaptation of Venable’s CCM4DSR approach, which differentiates the problem space from the solution space. The former focuses on the problem, stating what is undesirable about it and what its causes are while the latter focuses on its solution, specifying what benefits would accrue from solving the problem, what causes of the problem might be reduced or eliminated to solve the problem, and how such problem causes might be reduced or eliminated. The solution space can be derived by reversing the statements made in the problem space. For that purpose, statements in the problem space include both the undesirable and the desirable effects of an element, normaly separated by an ellipsis (…); for example: “High … low mental effort”. This way, the solution space counterpart can be derived: “Low … high mental effort”.
The highlighted part of the mind map will help you to perform a root cause analysis.
Once having understood the problem, it is time to start thinking about solving it. All the knowledge acquired during the problem analysis serves as a basis for effectively solving it, and the root cuases identified can be transformed as a requirements for the purposeful artefact and then used to evaluate it.
